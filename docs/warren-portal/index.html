📖 Quick Git Guide (the only 3 things you need)

Using VS Code's Built-In Git (no terminal needed!)

1

📥 Get the latest code (Pull)

Before you start working, always grab the latest changes.
In VS Code: click the Source Control icon (the branch symbol on the left sidebar) → click the ⋯ menu (three dots) → Pull.
Or just press Cmd + Shift + P, type Git: Pull, and press Enter.

2

💾 Save your work (Commit)

After making changes, you need to "save" them to Git.
In the Source Control panel: you'll see your changed files listed. Click the + next to each file (or click + next to "Changes" to add all). Type a short message describing what you did (e.g., "Added goblin spawn logic"). Click the ✓ Commit button.

3

📤 Share your work (Push)

After committing, push your changes so Jose can see them.
Click the ⋯ menu → Push. That's it! Your code is now on GitHub.
Or press Cmd + Shift + P, type Git: Push, and press Enter.

🆘 Something Went Wrong?

Don't panic! Git is confusing even for professionals. Here's what to do:

1. Don't delete anything — your code is safe.
2. Take a screenshot of the error message.
3. Text Jose with the screenshot — he'll fix it in 5 minutes.

If you're stuck and Jose isn't available, you can always start fresh by opening the Browser Editor at the top of this page — it always has the latest code.
