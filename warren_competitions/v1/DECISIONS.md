# Decisions checklist (to do with Warren before Month 1)

Short list. Run through it in one sitting. Takes ~30 minutes.

## 1. Game or app?

- [ ] **App** (Option 1 Counsel, Option 6 Muse) → primary target is
      Congressional App Challenge + MIT THINK. Smaller scope. Safer
      shipping.
- [ ] **Game** (Option 2 Kindred Village, Option 3 NPC Republic,
      Option 4 Lighthouse, Option 5 Quorum) → primary target is Games
      for Change + CAC (as a web game). Reuses Warren's Unity skills.
      Bigger scope; higher ceiling.

## 2. Which concept?

- [ ] Option 1 — Counsel (philosopher advice, web app) ⭐
- [ ] Option 2 — Kindred Village (cozy strategy sim) ⭐
- [ ] Option 3 — NPC Republic (sandbox god-game)
- [ ] Option 4 — Lighthouse (narrative puzzle)
- [ ] Option 5 — Quorum (debate strategy)
- [ ] Option 6 — Muse (creative prompt app)

See `CONCEPTS.md` for details on each.

## 3. Scope guardrails

- [ ] Hold weekly pace: **4–8 hours/week**, no more during the school
      year.
- [ ] MVP is **3 features**. Everything else is "nice to have" and only
      unlocked if ahead of schedule.
- [ ] Every month ends with a playtest — even if the game is ugly.

## 4. Competitions to commit to now

- [ ] Register for the **Congressional App Challenge (CAC)** the day
      the portal opens this cycle (May 1, 2026). Parent creates the
      account at congressionalappchallenge.us since Warren is a minor.
- [ ] **MIT THINK Scholars:** grades 9–12 only — Warren (8th grade) is
      **not eligible this cycle.** Park it as a Year-2 target (his
      9th-grade year, Nov 2026 portal open → Jan 2027 deadline).
- [ ] **Games for Change (G4C) Student Challenge:** the March 2026
      cycle has already closed. Next cycle opens late 2026 with a
      ~March 2027 deadline — Year-2 target if Warren picks a game.
- [ ] If a hackathon weekend looks interesting during Months 2–4,
      register Warren for one **Major League Hacking (MLH) Junior**
      event (parent consent required).

## 5. Tooling

- [ ] Repo on GitHub (public, README with the pitch).
- [ ] Copilot CLI + spec-kit for the initial spec.
- [ ] Claude Code or Copilot CLI with ralph-loop for iterative build
      (nights/weekends autonomous runs, parent reviews diffs in the
      morning).
- [ ] Sub-agents / skills for specialized tasks (e.g., a "rubber-duck
      reviewer" agent for Warren to bounce feature ideas off before
      implementing).
- [ ] One free hosting target: **Vercel** if web app, **itch.io** if
      game.

## 6. The one thing judges always look for

Write Warren's answer to this in one sentence, pinned to the top of the
repo README:

> *"I built this because ________."*

If Warren can't fill that blank honestly, switch concepts.

## 7. The "do not" list

- [ ] **Do not** claim the app is therapy or medical advice.
- [ ] **Do not** build two projects in parallel.
- [ ] **Do not** skip the 3-minute demo video — it is roughly half of
      the CAC rubric even though it doesn't look like it.
- [ ] **Do not** use uncited quotes from real people or copyrighted art
      without proper attribution/licensing.
